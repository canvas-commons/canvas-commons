# Authoring plugins

Plugins are a way to extend the functionality of Canvas Commons.

Canvas Commons consists of two parts running in separate environments: a
[Vite](https://vitejs.dev/) development server running on Node.js and an editor
running in the browser. Browser-related code can be split further into two
parts:

- Runtime - all the logic necessary to play the animation, such as the `Player`
  instance, Scene definitions (including the actual source code of your
  animation), etc.
- Editor - the interface that lets you preview the animation, edit the settings,
  etc.

# Plugin examples

You can take a look at our [WebCodecs plugin][webcodecs-plugin] for an example
of a fully-fledged plugin. Its runtime portion provides the editor with a Video
[exporter](/api/core/app/Exporter) that encodes the rendered frames into a video
in the browser using the WebCodecs API. The server portion then writes the
finished video to the output directory.

[webcodecs-plugin]:
  https://github.com/canvas-commons/canvas-commons/tree/main/packages/webcodecs
