# Video (WebCodecs)

An exporter that uses the browser's
[WebCodecs](https://developer.mozilla.org/en-US/docs/Web/API/WebCodecs_API) API
to turn your animation into a finished video. Encoding happens entirely in the
browser, so there's no native dependency to install and no need to ship frames
to an external tool.

The exporter is still relatively new and may not have all the features you need.
If you have an idea for how to improve it, feel free to [open a feature
request][issues] on GitHub.

## Installation

:::info

The exporter runs in the browser and requires no external software. Make sure
your browser supports the [WebCodecs API](https://caniuse.com/webcodecs) and the
codec you want to export.

:::

### New project

When [scaffolding a new project](/docs/quickstart#creating-a-new-project), you
can select _Video (WebCodecs)_ in the list of available exporters. This will
automatically install the required dependencies and add the necessary
configuration.

### Existing project

If you want to install the exporter in an existing project, run the following
command:

```bash
npm install --save @canvas-commons/webcodecs
```

And then configure it in your `vite.config.ts` file:

```diff
  import {defineConfig} from 'vite';
  import canvasCommons from '@canvas-commons/vite-plugin';
+ import webcodecs from '@canvas-commons/webcodecs';

  export default defineConfig({
    plugins: [
      canvasCommons(),
+     webcodecs(),
    ],
  });
```

## Configuration

After selecting the Video (WebCodecs) exporter in the Video Settings tab, the
following configuration options will become available:

### Video codec

The codec used to encode the video. H.264/H.265 require even dimensions, while
VP8/VP9/AV1 also allow odd ones. The browser must support the chosen codec for
the export to succeed.

### Video quality

A quality preset (from _very low_ to _very high_) that controls the target
bitrate. Higher presets produce larger files with better visual fidelity.

### Include audio

When enabled, the audio track will be included in the rendered video. This
option is only available if you've [set up audio](/docs/media#audio) for the
current project.

### Audio codec

The codec used to encode the audio track. If the browser cannot encode the
chosen codec, the exporter falls back to another supported one.

### Fast start

When enabled, the video will be rendered with the Fast Start option. It's an
optimization for videos that will be served over the web. It rearranges the
video data in the file, allowing the video to begin playing before it has
finished downloading.
